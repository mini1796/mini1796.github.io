SkillMedha CMS Platform

A complex, multi-portal system for students, colleges, and corporate partners.

Problem

This massive project required three distinct user portals (Students, Colleges, Corporates). The main problem was a massive inconsistency and slow development cycle caused by lacking a unified Design System that could support complex dashboards for admins while remaining simple enough for students.

Research

I audited over 150 existing screens across the platform. I identified 12 unique, inconsistent button variations, 8 competing table styles, and a complete lack of typographic hierarchy. The frontend team was spending 40% of their sprints just rewriting CSS overrides to match ad-hoc design files.

User Flow

The flow starts with Role Based Access Control (RBAC): User Logs in → System detects Role (Student/College/Corporate) → Routes to specific portal view → Renders specific dashboard using the newly unified component architecture bound to that role's permissions.

Wireframes

The wireframing phase ignored high-fidelity looks and instead focused entirely on structural component hierarchy. I mapped out the atomic building blocks (inputs, buttons) before assembling them into complex organisms (data tables, modal forms).

Final UI

The final deliverables were not just screens, but a highly organized Figma Library utilizing full Auto-Layout, Component Variants, and deeply nested Figma Typography/Spacing Variables that bridged the gap elegantly between UX and Code.
